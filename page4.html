Op donderdag 4 oktober was een van de opdrachten op mijn stage was het netwerk van het bedrijf visueel voorstellen. Hierbij ben ik veel onbekende termen tegengekomen zoals MPLS. MPLS is eigenlijk een toestel dat in verbinding staat met verschillende 'sites' van het bedrijf hierover loopt dus vooral corporate data. Het bedrijf werkt ook met een software genaamd Citrix. Citrix maakt het mogelijk mensen van bijvoorbeeld Nederland te laten werken op een computer die in het datacenter in Beerse draait. De visuele voorstelling mocht ik maken met een nieuwe software, die heel hard lijkt op draw.io. De software "Visio" is een software ontwikkeld door Microsoft. Op de tekeningen zou ik voor mijn case study verschillende situaties willen voorstellen en uitleggen hoe de verschillende data door het netwerk loopt.

De visuele voorstelling die ik gemaakt heb:

Situatie 1: Het hele netwerk werkt

De corporate data van PC 1 zal via de access switch geleidt worden naar de Fortinet Firewall omdat hij niet langs de MPLS kan gaan sinds deze down is. De data zal door een VPN tunnel moeten gaan die van remote Firewall naar central Firewall loopt, over internet. Eens de data aankomt bij de remote Firewall wordt hij verder geleidt naar de core switch die zo naar de gewenste server gaat. De internet data kan nog steeds via de firewall rechtsreeks naar het internet geleidt worden. Voor de corporate data van PC 2 is deze situatie niet echt van toepassing omdat er geen MPLS aanwezig is. De internet data wordt op dezelfde manier verstuurd als bij de vorige situatie.

Situatie 2: MPLS Down

De corporate data van PC 1 zal via de access switch geleidt worden naar de Fortinet Firewall omdat hij niet langs de MPLS kan gaan sinds deze down is. De data zal door een private VPN tunnel moeten gaan die van Firewall naar Firewall loopt, over internet. Eens de data aankomt bij de fortinet Firewall wordt hij verder geleidt naar de core switch die zo naar de Citrix server gaat. De internet data kan nog steeds via de de firewall rechtsreeks naar het internet geleidt worden. Voor PC 2 zal de corporate data via de access switch naar de firewall geleidt worden en zo via nog een andere private VPN tunnel naar de andere Fortinet firewall geleidt worden waardoor we terug bij de core switch terechtkomen en zo de data kunnen verder sturen naar de Citrix server. De internet data wordt op dezelfde manier verstuurd als bij de vorige situatie.

Situatie 3: Internet Down

Voor PC 1 zal de corporate data net zoals bij situatie 1 gewoon via de MPLS de weg afleggen naar de server. De internet data zal naar de access switch gaan, zo naar de remote fortinet firewall, deze firewall zal dan de data sturen naar waar het moet. In dit geval zal het terug langs de access switch gaan, zo naar de router en dan naar de MPLS. De MPLS stuurt de data door naar de core switch en dan naar de gewenste server en zo naar het internet. De data van PC2 kan niet verstuurd worden als het internet down is.
